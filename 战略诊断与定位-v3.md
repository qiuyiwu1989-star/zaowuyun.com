## 0 · 一页结论(TL;DR)

**身份(对内真相):** 造物云 = 一家**产品杠杆型的企业 AI 落地公司**——不靠人头做项目,而是把每一次落地都跑在自研产品上,越做越厚、越做越快。主推产品 = **ZaoCode**。落地服务围着 ZaoCode 转:产品打头、服务垫后(Palantir 式"有产品的解决方案公司")。

**产品(对外主角):** ZaoCode(`ai.zaowuyun.com`)= **企业自己的 AI 工作台 / 中台**。底层内核是 coding agent + 多模型中立路由,但**绝不按"coding agent / AI 编程工具"来卖**(那是 Cursor/Claude Code 的绞肉机,基座层商品,打不赢巨头)。

**品类豪赌(最锋利的一刀):**
> **ZaoCode 不是"给程序员的 coding agent",是"给业务专家的 AI 造物平台"——让不会写代码的业务人员,自己造出专业的 AI 应用产品。**
> 全世界的 coding agent 默认建造者是程序员;ZaoCode 把建造者换成业务专家。这不是同赛道做得更好,是换赛道。它正中模型盲区(模型擅长"怎么造"、不擅长"造什么/什么叫好",而业务专家恰恰握着后者)。

**护城河:** 设计×商业的领域判断(模型抄不走)× 应用/插件/skill 中心带来的复用复利(Process Power)× 私有化前向部署形成的切换成本。盯**一个**健康指标:复用率。

**第一刀垂直:** 消费品 / 产品全生命周期先做穿。自营应用(aha、colorforge、promptcard…)从"业务主线"降为"**我们就是用 ZaoCode 造出这些应用的**"活体战绩。

**必须守住的纪律:** 应用/插件/skill 中心是企业**内部**的造物工厂(管道,深化切换成本),**不是**第三方上架抽成的开放平台/生态(双边网络效应、冷启动悖论)。开放生态 = 三年后的期权,现在不喊。

## 2 · ZaoCode 是什么,该怎么定位

### 2.1 它实际是什么(现状)
- 网页版 coding-agent 产品(`zaocode` 服务,内网 8096,nginx 反代 `ai.zaowuyun.com`)。
- **多模型中立路由**:`default / codex / xiaomi`,默认把 Anthropic 系模型映射到 `mimo-v2.5-pro`(小米 MiMo)。可接国产模型、不锁一家。
- 已挂 `/app`、`/aha/`,旁边一串兄弟子站(sb.ai 沙箱、zz-ai、shennao、dam 检索)。
- 后续路线:**应用中心、插件中心、skill 库**——让业务人员自助造专业 AI 应用。

### 2.2 生死叉:同一个二进制,两种打法
| | 打法 A(自杀) | 打法 B(成立 · 已采纳) |
|---|---|---|
| 定位为 | 一个 coding agent / AI 编程工具 | 企业自己的 AI 工作台 / 中台 |
| 建造者 | 程序员 | **业务专家** |
| 对手 | Cursor / Claude Code / Cline / 国产克隆(基座层绞肉机) | 无正面对位的巨头(它们锁自家模型、要翻墙、不私有化) |
| 结局 | 和巨头硬刚基座层,无模型/资本优势,必死;且暴露"套壳" | 卖内核**之上**那一层(部署+中立+全员造应用+领域模板),立得住 |

**关键判断:同一个产品,pitch 的海拔决定生死。** ZaoCode 现有的 `mimo`/多 provider/内网架构,恰好是打法 B 的弹药、A 用不上。所以站 B。

### 2.3 ZaoCode 卖的四个支柱(打法 B 的实体内容)
1. **模型中立 / 可接国产**:mimo、Codex、Claude 随切,不绑一家,国产合规可用——巨头结构上做不到。
2. **私有化 / 数据不出门**:内网部署,国内企业(制造、消费品)买单的硬门槛。
3. **全员能造应用,不只程序员**:把 coding agent 从程序员工具,扩成全公司的造物手(详见 §3)。
4. **领域模板 / 判断沉淀**:消费品全生命周期的判断编进去,越做复用率越高(护城河,要主动灌)。

> 前三条现有架构已支持;第四条是要主动锻造的壁垒。

## 3 · 品类豪赌:给业务专家的 AI 造物平台

这是 v3 最锋利、也最该被反复打磨的一刀。

### 3.1 为什么这是"换赛道"而非"做得更好"
- 全世界的 coding agent 默认用户是程序员。ZaoCode 把建造者换成**业务专家**——一个品类边界的位移,不是同赛道竞速。
- 一句话主张:**不会写代码的业务人员,也能造出专业的 AI 应用产品。**

### 3.2 为什么它正中要害(踩在 v2 研究结论上)
1. **正打模型盲区**:模型擅长"怎么造",不擅长"造什么 / 什么叫好"(a16z、红杉双源)。业务专家手握领域判断与品味;ZaoCode 让这份判断**不经过程序员**就变成能跑的应用——这是真正的解锁。
2. **就是复利引擎**:应用中心 / 插件 / skill 库 = Process Power。业务员每造一个 skill,就给平台加一块可复用资产,复用率上行;全公司的 skill 长在 ZaoCode 里,切换成本自然变厚。
3. **接住"AI 落地"**:落地难,是因为只武装了程序员、业务流没动。全员能造应用 = 业务流真正 AI 化、老板感觉得到价值。

### 3.3 必须守住的纪律(别把这刀划伤自己)
- **现在做(对)**:企业**内部**的应用工厂——客户业务员造自己公司用的应用。这是**管道**,深化切换成本。
- **以后的期权(别现在喊)**:第三方上架、互调、抽成的开放生态。那是双边网络效应、冷启动悖论,三年后、且要用"自产填满"破冷启动才碰。
- 因此:**skill / 插件中心 = "造物的复利账本",不是"我们要做平台"的招牌。** 全站不出现"平台 / 生态 / 应用市场 / 网络效应"。

## 4 · 护城河(7 Powers 收敛版)

| 力 | 来源 | 判断 |
|---|---|---|
| **Cornered Resource** | 浙大设计血脉 + 12 年造新物种的设计×商业判断 + 消费品行业纵深 | ✓ 真有,模型抄不走"造什么/什么叫好" |
| **Process Power** | 应用/插件/skill 中心的复用复利;每次落地沉淀可复用资产 | ✓ 主壁垒,要主动锻造;盯复用率 |
| **Switching Costs** | 私有化前向部署 + 全公司 skill/应用长在 ZaoCode 里 | ✓ 可建,越用越离不开 |
| **Counter-Positioning** | "给业务专家造,不给程序员造";中立不锁、可私有化、接国产 | ◐ 有戏,对 Cursor/Claude Code 是结构性反定位 |
| **Scale / Branding** | 仅在消费品垂直内复利;创始人 IP + 浙大攒中 | ◐ 垂直内有效,需时间 |
| **Network Effects** | —— | ✗ 没有,继续别讲平台/生态 |

→ 护城河组合 = **判断(Cornered Resource)× 造物复利(Process Power)× 私有化切换成本**。三者都要求"主推一个深嵌客户的产品",而非"远远卖一套工具"。

## 5 · 公司定位与商业模式

### 5.1 一句话
**对内:** 造物云 = 产品杠杆型企业 AI 落地公司;主推 ZaoCode(给业务专家的 AI 造物平台),围它做落地交付与共创,在消费品垂直筑墙。复用率是唯一健康指标。

**对外:** 造物云 —— 让你公司的业务人员,自己造出专业的 AI 应用。
> ZaoCode 部署在你的环境、数据不出门、模型中立不锁;你的业务专家不写代码也能把判断变成能跑的 AI 应用,全公司的 skill 越攒越厚、越用越离不开。我们自己就用它造了一批应用——先拿自己练,才敢拿你的生意来跑。

### 5.2 海拔模型(产品打头版)
```
   ┌──────────────────────────────────────────────┐
 ④ │ 判断层  "造什么·什么叫好"——业务专家的品味(模型盲区) │ ← Cornered Resource
 ③ │ 产品层  ZaoCode = 企业 AI 工作台 + 应用/插件/skill 中心 │ ← 主推,Process Power 圈地
 ② │ 服务层  围 ZaoCode 的落地交付 / 共创(产品垫后的服务)   │ ← 现金流,land-and-expand
 ─ ├──────────────────────────────────────────────┤
 ① │ 自营应用 aha/colorforge/promptcard…(我们用它造的)    │ ← 活体战绩,不当主线
 ⓪ │ 基建层  coding agent / 大模型(骑上去,不拥有不炫耀)    │ ← 商品,放手
   └──────────────────────────────────────────────┘
   贯穿全栈的轴 = 设计×商业×AI 的造物方法论 + 浙大血脉信誉旗
```

### 5.3 商业模式
- **产品收入(主,逐年爬升)**:ZaoCode 订阅 + 私有化部署 license;应用/插件/skill 中心带动用量。
- **服务收入(垫后,现金流)**:落地交付、领域模板定制、培训(把客户业务员教成"AI 造物者")、共创。
- **land-and-expand**(Palantir 式):小楔子部署 → 业务员造出第一批应用见效 → 叠用例 → 收入结构从服务漂向产品。
- **纪律**(继承 v1):确定性营收穿越 J 曲线;活得久>跑得早;复用率逐季上升。

### 5.4 第一刀楔子(JTBD)
- 客户雇你完成的活:"我有业务、有判断,但我的业务专家不会把判断做成能跑的 AI 应用;买的一堆 AI 工具串不起来。"
- 切口:消费品的**产品定义/内容/视觉/上市增长**等高判断环节,让其业务团队在 ZaoCode 上自助造应用。
- 标杆:品唤(共创样板)+ 自营 1–2 个深做(战绩)。

## 8 · 待拍板与缺口

### 待邱懿武拍板
- **A. 两站分工**:`ai.zaowuyun.com` 做纯产品站、造物云主站做公司站,认不认?
- **B. ZaoCode 对客户的最小可卖单元**:先卖"私有化部署整套"还是"先 SaaS 试用、再私有化"?(影响定价与销售动作)
- **C. "业务专家造应用"的第一批模板**:先做哪 1–2 个消费品场景的应用模板当样板(决定 Demo 怎么搭)?
- **D. 自营应用的对外口径**:全部统一成"我们用 ZaoCode 造的战绩",还是保留个别独立品牌叙事?

### 缺口(可补,不挡定稿)
1. **竞品实测**:Dify/扣子/字节等"低代码 AI 应用"与 ZaoCode"业务专家自助造"的真实差异与采纳门槛(决定品类话术锐度)。
2. **私有化部署的单位经济**:每客户部署/运维成本 vs 订阅,毛利能否撑住产品估值故事。
3. **"业务专家真能造出专业应用"的真实成功率**:需要 1–2 个非程序员用户的实证(产品的核心承诺,必须自证)。
