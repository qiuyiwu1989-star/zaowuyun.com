## 0 · 一句话定位

**对内(真相):**
> 造物云 = 一家护城河在「认知层方法论」的**高 IP 产品化咨询公司**(管道,不是平台)。用浙大设计系血统的产品化能力 × coding agent 的交付力,把企业模糊的经营判断,做成一套**会自己进化的 AI 中台**;第一刀锁死「消费品 · 产品全生命周期」。靠确定性营收穿越 J 曲线,平台化是三年后的期权。

**对外(给客户):**
> **造物云 —— 让企业拥有一个会自己进化的 AI 中台(AI 时代的 ERP)。**
> 把你整条业务流程跑在上面,部署在你自己的环境、数据不出门;以 coding agent 为大脑,用你每天产生的文件和技能直接学习(不做没人用的知识库),**越跑越懂你的公司、越用越离不开**。
> 矛尖:**别人给你一堆串不起来的 AI 工具、或一个没人用的知识库;造物云把你整个公司,变成一个越用越聪明的 AI 中台——你自己的那一个。**

# 第一部分 · 对内(我们到底是什么)

## 1 · 身份判定:高 IP 的产品化咨询(管道,不是平台)

- 用 InnoLab「平台 vs 管道」判定:造物云卖的是「我们给你**交付 + 部署 + 培训**一套中台」= **管道(Pipeline)**,企业自己生产价值、线性增长。
- **不是平台。** 老官网「中台之上长出产品矩阵 + 开放平台 + 生态」是把自己误装成微信/淘宝式平台——但没有双边市场、没有网络效应,是范畴错误,也是「讲故事陷阱」的根。
- 参照系 = **Palantir 的 FDE 模型**(有产品的咨询公司)、AI 时代的「Services-as-Software」。对外用**医生**比喻(出诊=交付 / 治疗=部署 / 教家属护理=培训 / 背后是治疗方法=SOP)。
- **承认这个身份是优势,不是降格**:它给你确定性营收(治你们的疤),AI 让每人=过去十人(why now),贴合你们的真实 DNA。

## 2 · 核心产品点

**唯一产品 = 一套会进化的、属于企业自己的 AI 中台,把整条业务流程(立项→开发→生产→运营→销售→数据→复盘→再立项)跑在上面。**

第一刀(楔子)= **实体产品 / 消费品公司的「产品全生命周期 · 数字项目经理」**(IPD-lite,可配置)。
- 理由:"只有项目是从头串到尾的"——它天然串起整个公司,「串起来」的价值最大;且正压在你们最深的行业(苏泊尔/品唤/消费品/跨境电商)。
- 活例:配色管家(产品 + 配色评审录音 → 自动生成产品定位文档/表格)。

中台的四件套(对每个客户)= **文件(存中台、AI 大权限调度)+ 大脑(Claude/Codex,统一买账号、不翻墙、内网部署)+ 手(应用/插件,员工自己也能开发)+ 自动化任务(AI 自己顺流程跑、通宵跑)**。

## 3 · 护城河:认知层,不是技能层

用 InnoLab「护城河迁移」(技能→认知→价值)盘点:

| 优势 | 层级 | 判断 |
|---|---|---|
| 会用 Claude Code/Codex、会写插件、会搭中台 | **技能层** | ⚠️ 贬值最快(AI 10 秒到 80 分),**不是护城河,对外别炫** |
| **浙大设计系近 20 年批量培养数字产品 PM/设计师的方法论传承** + SOP + InnoLab + 把判断编码成 skill/模板 | **认知层** | ✅ 真护城河(10–20 年),机构级、难抄、可育人 |
| 浙大孵化血统 + 医生品牌 + 标杆案例口碑 | **价值层** | ✅ 终身 |

- **护城河必须锚在认知层。** 你自己的方法论原话:"方法论体系是 AIGC 时代真正的护城河。"
- **浙大血统的战略价值**:① 把"批量复制交付人(开发者+部署者+培训师)"从待证变成本来就会(这条血脉本就是育人工厂,杀 key-man 风险);② 认知层护城河有机构来源,不再是"创始人个人=不可让渡"。
- **诚实注脚**:这 20 年是「互联网思维」那代肌肉。可直接迁移 = 产品化能力 + 设计思维 + 批量育人(coding agent 时代的稀缺互补品);**须主动做互联网→AI 思维跃迁,别躺在"培养过互联网 PM"上**。

## 4 · 复利引擎:数据×场景飞轮(运行即进化)

- 飞轮:**场景 → 数据 → 模型 → 更好的场景**。起点是场景,不是数据。
- **反知识库/反 RAG**(InnoLab 飞轮方法 + 客户原话双重背书):"不是有了数据就有 AI,是有了场景才有数据";"做知识库纯扯淡、没人用"。造物云的反命题 = **文件即记忆、skill 直接读、运行即进化**。
- 楔子用飞轮三条件检验,全中:消费品产品全生命周期 = 数据丰富 ✅ + 决策频繁 ✅ + 反馈闭环 ✅。
- 冷启动靠外力:**品唤深度共创 = 人工冷启动种子**。

## 5 · 商业模式:确定性营收 → 浅 J 曲线 → 跑得久

- 报价结构:**便宜楔子(1999 试用)→ 2 万/年订阅 → 定制 10–30 万 upsell**。每个行业头部先走一圈。
- 用 InnoLab「J 曲线」:老路(面向投资人讲故事)= 深 J 曲线,你只能亏两年、被「能亏二十年」的酷家乐收割;**产品化咨询 = 浅 J 曲线**(交付即收钱),让你活过谷底。
- 原则:**活得久 > 跑得早。** 用确定性营收把 J 曲线压浅。
- ROI 话术(客户自己的语言):"投 1 块钱在 AI 上,增加 100 块营收";省下大量会议工时(140 人时/产品 × 300 产品/年)。

## 6 · 三条纪律(生死线)+ 唯一健康指标

产品化咨询做好(天花板)还是做砸(坟墓/外包),全看这三条:

1. **「一套中台」必须真共享——盯「复用率」这一个数。** 每个新单里"模板复用 vs 净新定制"的比例,必须逐季上升。**这是唯一要盯的健康指标。**
2. **课程/SOP 不是副业,是「复制交付人」的工厂 + 杀 key-man 风险的解药。** 一身三任:训练管线 + 获客漏斗 + 编码判断的护城河。
3. **主动让「产品/IP 收入占比」随时间爬升**,从咨询毛利漂向产品毛利(Palantir 路径)。

**平台化 = 三年后的期权**:等组件/技能库开放给第三方上架、客户互相调用,才升级成平台,且冷启动用「自产」策略(自己先把组件填满)。现在强行做平台 = 死在冷启动悖论。

## 6.5 · 待拍板(唯一还需你确认的)

- **可复用内核边界**:第一刀是否**锁死**「消费品 · 产品全生命周期」一个模板,先做深做透、做成能复制、能出标杆案例的拳头?(强推:锁死)
- 其余(管道不是平台、护城河在认知层、确定性营收穿越 J 曲线、医生对外框架)本轮已收敛,如无异议即定稿。

# 第二部分 · 对外(给客户怎么讲)

## 7 · 主张与话术

- **H1**:你自己的 AI 中台 —— AI 时代的 ERP。
- **副标**:把整条业务流程跑在上面,部署在你的环境、数据不出门;以 coding agent 为大脑,越跑越懂你的公司。
- **矛尖**:别人给你一堆串不起来的 AI 工具、或一个没人用的知识库;造物云把你整个公司,变成一个越用越聪明的 AI 中台。
- **买家 = 决策者(老板/一号位)**,不是使用者。他买的是「敢拍板的依据 + 风险可控」。
- **痛点开口**:你有 ERP/CRM,但业务不在上面跑;你买了一堆零散 AI 工具,串不起来、价值不大;你想做知识库,几百万砸进去没人用。

## 8 · 怎么运作(给客户的机制故事)

- **中台是你自己的**:部署在你的服务器/内网,数据/算力/API/文件/通信全不出门 → 合规;统一买账号、不用翻墙、打开就用 → 解决 access。
- **不做知识库**:文件丢进文件夹,AI 直接读;工作流跑一遍就进化一次。失败案例进「失败库」,下次自动预警不重犯。
- **风险阀门式人机协作**:人只守高风险闸(合同/法律/成本/信用),低风险环(测款、内容、官网维护)让 AI 自己闭环跑——"AI 给人打工"(钉钉吴昭式组织样板)。
- **多智能体**:Claude 思考 / Codex 执行 / 龙虾通知,你不用学客户端,跟数字助理说句话就行。

## 9 · 楔子与标杆案例

- **第一刀**:给做实体产品的公司,一个从立项到复盘、从头串到尾的「数字项目经理」。
- **标杆案例(必须先做出来)= 品唤(youforyou)**:深度共创、付费样板。医生靠案例攒口碑,这是第一个治愈案例。
- **团队画像**:每个交付者 = **AI 时代的产品经理/设计师**(coding agent 当手,人出脑、审美、判断)。

## 用词纪律

- **要说的词**:你自己的 AI 中台 / AI 时代的 ERP / 越用越离不开 / 数据不出门 / 会进化 / 数字项目经理 / 医生(治疗方法)。
- **不能说的词**:咨询/外包(低位,对内才用)、操作系统/判断 OS/复利回路(愿景/投资人词)、curl/MCP/开发者那套(A 味)、"我们会用 Claude Code"(技能层,贬值)、平台/生态(还没到)。
